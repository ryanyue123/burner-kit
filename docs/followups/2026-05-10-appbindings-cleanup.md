# Follow-up: Replace `AppBindings` with declaration-merged `Env`

**Status:** open
**Created:** 2026-05-10
**Originated from:** [Confirmation code autofill plan](../superpowers/plans/2026-05-10-confirmation-code-autofill.md), Task 3.

## Why

The worker has two parallel types describing the same bindings:

- **`worker-configuration.d.ts`** â€” auto-generated by `wrangler types` from `wrangler.toml`. Declares a global `Env` interface.
- **`AppBindings`** in `apps/worker/src/middleware.ts` â€” manually maintained type used throughout the app.

These overlap. Every time we add a binding (D1, AI, Queue, etc.) we have to remember to update both. Drift is inevitable.

Cloudflare's documented modern pattern is **single source of truth in `wrangler.toml`** â†’ `wrangler types` regenerates the `Env` interface â†’ app code references `Env` directly. The escape hatch for the cases the auto-gen can't see (secrets, narrowed generic types) is **declaration merging** on the global `Env`.

References:

- [Write Cloudflare Workers in TypeScript](https://developers.cloudflare.com/workers/languages/typescript/)
- [Improving Workers TypeScript support Â· Cloudflare Blog](https://blog.cloudflare.com/improving-workers-types/)

## Target state

```ts
// apps/worker/src/types/env.d.ts (new)
declare global {
  interface Env {
    BETTER_AUTH_SECRET: string; // secret, not in wrangler.toml
    OTP_EXTRACTION_QUEUE: Queue<{ messageId: string }>; // narrows the generic Queue
  }
}
export {};
```

`AppBindings` gets deleted. All references to `AppBindings` (function signatures, imports) become `Env`.

## How

1. Create `apps/worker/src/types/env.d.ts` with the merged interface above.
2. Confirm `tsconfig.json` already includes `worker-configuration.d.ts` so `Env` is in scope globally (it should be â€” current setup already relies on this).
3. Delete `AppBindings` from `apps/worker/src/middleware.ts`.
4. Replace every `AppBindings` reference with `Env`. Likely call sites:
   - `apps/worker/src/index.ts` (default export's `fetch`, `queue`, `scheduled` params)
   - `apps/worker/src/middleware.ts` (`makeAuthMiddlewareLive`)
   - `apps/worker/src/runtime.ts` (`makeAppLayer`)
   - Anywhere else that imports `AppBindings`
5. `pnpm cf-typegen` to ensure the generated `Env` is current.
6. `pnpm typecheck` â€” should pass.
7. `pnpm dev` â€” smoke test the worker still starts and serves requests.

## Why not in the original PR

The confirmation-code-autofill PR is a feature. This is a binding-type refactor that touches every file taking `env`. Mixing them makes both harder to review. Land the feature, then come back for this as its own small PR.

## Acceptance

- `AppBindings` no longer exists anywhere in the codebase.
- All worker entry points and handlers take `env: Env`.
- A single `.d.ts` file (new) lists everything not auto-genned.
- `pnpm typecheck` and the worker dev server both work.
